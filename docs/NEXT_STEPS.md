# Next Steps

## Immediate Build Plan

1. Choose the project name.
2. Create the mobile-style web prototype.
3. Build the first app screens:
   - Onboarding
   - Breed picker
   - Dog profile setup
   - Today
   - Diary
   - Care
   - Calendar
   - Profile
4. Use sample data for a Great Pyrenees dog.
5. Add friendly animations and mobile bottom navigation.
6. Review the prototype flow and adjust the product direction.

## Prototype Content Needed

For the first clickable version, we can use placeholder content:

- Dog name: Bear
- Breed: Great Pyrenees
- Birthday: April 10, 2022
- Weight: 92 lb
- Personality: gentle, mountain-loving, stubborn, loyal
- Example diary entry: waterfall hike
- Example reminders: vet appointment, heartworm medication, food refill
- Example care records: vaccines, medication, weight check

## Decisions To Make Soon

- Final project/app name
- Visual style direction
- Whether the prototype should be Next.js or Vite/React
- Whether to start with local mock data or a small database immediately
- Whether GitHub repository should be public or private

## Not In The First Prototype

These are important, but should come after the initial prototype:

- User login
- Multiple pets
- Shared caregiver access
- Real push notifications
- Email notifications
- Calendar sync
- GPS walk tracking
- DNA/genetic report imports
- Native Android build

## First Prototype Success Checklist

- The app feels cute and companion-like.
- The care information feels organized and trustworthy.
- A user can understand the daily workflow quickly.
- The diary feels like a real place to keep pet memories.
- The future roadmap is visible but does not overwhelm version one.
